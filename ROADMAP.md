# 🗺️ Roadmap

El programa se construye por fases. El **currículo escrito** (README completo por cada clase)
es la base y el primer entregable.

## Fase 1 — Currículo escrito (en curso)

- [x] Diseño del currículo: 18 partes, ~292 clases, numeración secuencial.
- [x] Estructura de carpetas + índice maestro (`classes/README.md`).
- [x] **Parte 0 — Fundamentos y prerrequisitos** (clases 001–025): README de parte + 25 clases completas.
- [x] **Parte 1 — Motores 2D y tu primer juego jugable** (clases 026–045): README de parte + 20 clases completas.
- [ ] Parte 2 — Desarrollo 3D (clases 046–067).
- [ ] Parte 3 — Física y matemáticas aplicadas (clases 068–085).
- [ ] Parte 4 — Gráficos, shaders y rendering (clases 086–107).
- [ ] Partes 5–17 — resto del programa (IA, audio, multijugador, diseño, arte, plataformas, web, VR/AR, optimización, tooling, publicación, capstones).

## Fase 2 — Laboratorios ejecutables (planificada)

- [ ] Proyecto base de Godot por parte, versionado y clonable.
- [ ] Lab **plataformas 2D** completo (Parte 1) como repositorio jugable.
- [ ] Lab **3D third-person controller** (Parte 2).
- [ ] Lab **shaders** con ejemplos ejecutables (Parte 4).
- [ ] Lab **multijugador** cliente-servidor mínimo (Parte 7).

## Fase 3 — Material complementario (planificada)

- [ ] Guías **PDF** imprimibles por clase.
- [ ] Presentaciones **PPTX** por clase.
- [ ] Assets de práctica (sprites, tilesets, sonidos con licencia libre).

## Fase 4 — Portal y evaluación (planificada)

- [ ] Sitio web navegable del currículo (GitHub Pages).
- [ ] Autoevaluaciones interactivas por parte.
- [ ] Seguimiento de progreso de las clases (localStorage).
- [ ] Rutas guiadas por rol.
